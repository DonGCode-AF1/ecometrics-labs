### 💭 AI 核心思考与设计决策 (AI Design Decisions)

#### 1. 物理目录结构设计
* **决策**：在工作区根目录下独立划分 `empirical-playground` 作为计量实验沙盒。不干扰原官方课件的 `empirical-methods-course` 和交互大看板 `interactive-simulations`。
* **原因**：这保证了项目边界清晰，其他人克隆 `empirical-playground` 时能作为独立的 Python-Jupyter 工作区进行运行。

#### 2. Jupyter 交互技术选型
* **决策**：采用 `ipywidgets` (滑块与选择器) 结合 `matplotlib` 渲染动态图像，而不使用复杂的 Shiny 或 Dash。
* **原因**：`ipywidgets` 可以完全嵌入在单个 `.ipynb` 笔记本中，轻量级、无需搭建复杂的本地 Web 容器，用户在 Jupyter Lab / VS Code 中双击即可体验，完美契合“一键配置、直观展示”的诉求。

#### 3. 语言风格调校 (去 AI 腔)
* **决策**：在所有实验的解读文档中，剔除多余的煽情、赞美、引申词汇（如“不是……而是……”、“极其深刻”、“惊人地”）。
* **策略**：
  * **用数理和逻辑推断说话**：解释为什么自相关会导致标准误被低估（因为自相关下，误差项包含了重叠的系统性冲击，OLS 将其视为独立样本，因而高估了有效样本量 $N$，进而低估了标准误差）。
  * **文字结构**：公式推导 $\to$ 模拟图表验证 $\to$ 核心思想启发（核心思想为“永远在政策分配层级聚类”）。

#### 4. 可追溯与接手机制
* **决策**：
  * 创建 `prompt-thoughts-log/` 文件夹。
  * 将用户的 Prompts 和模型的底层思考逻辑记录于本文件中 (`prompt_thoughts_archive.md`)。
  * 设立类似 Git commit-log 风格的物理文本文件 `operation_history.log`，记录操作人员和具体变更。
  * 在 `ai-coding-skills.md` 中增加“新 AI 助手一键配置与文档同步更新规范”，确保未来接手的 AI 能够实现自动化日志和 README 的同步迭代。

### 💭 AI 核心思考与设计决策 (AI Design Decisions)

#### 1. 蒙特卡洛（Monte Carlo）的直觉化降维教学
* **设计决策**：摒弃死记硬背的渐近大样本代数推导，改用**“计算机平行宇宙体检”**这一极具物理直觉的类比。
* **教学直觉**：
  - 用“造物主”的上帝视角向用户讲明白：我们在计算机内定死了政策真实效应为0（零假设 $H_0$ 为真）。
  - 然后模拟 1000 个平行宇宙。如果一个标准误公式是“诚实、正确”的，那么误诊率（拒绝率）应当且仅应当在 5% 左右。
  - 如果经典标准误拒绝率飙升至 45%，意味着它在 45% 的平行宇宙中“撒了谎”。

#### 2. 黑金玻璃拟物化网页实验室（Aesthetic Glassmorphism Dashboard）
* **设计决策**：放弃简单的 HTML 文本，采用顶尖现代前端 UI 设计规范：
  - **配色体系**：暗色背景，搭配磨砂玻璃面板，使用高度对比的暖金、靛蓝、翡翠绿、深绯红来对应各标准误类型。
  - **性能黑科技**：在前端用纯 JavaScript 实现高性能的 Within-Panel Panel OLS 面板双向去均值回归与组聚类方差估计。使数十万数据点的核密度计算与蒙特卡洛仿真在 JavaScript 内以毫秒级完成，拖动滑块时无需刷新网页、无需服务器，**SVG 图表进行流畅贝塞尔曲线过渡**。

#### 3. 经典与前沿图表的多维呈现
* **设计决策**：设计三大独立图表以实现直观具象化：
  - **拒绝率条形图**：展示三种标准误与 5% 名义警告线的偏离程度。
  - **t统计量经验分布拟合曲线**：使用 Epanechnikov 核密度估计（KDE）在 SVG 画布上动态绘制 $t$-统计量的经验 PDF 密度曲线，用户拉动 $\rho$ 到 0.8 时，能亲眼看到 OLS 的 t 分布曲线向两边“塌陷并长出巨大的胖尾”，越过 $\pm 1.96$ 的红色临界线。这以绝对完美的几何形式回答了“过度拒绝在数理上意味着什么”。
  - **置信区间覆盖体检图**：抽取 20 个平行宇宙的置信区间，若区间未包裹 0，则线段爆红（判定假阳性），直观展示区间覆盖率（Coverage Rate）随着自相关下降。

## 📅 [2026-05-22] 算法精度审计与小样本 Wild Cluster Bootstrap 退化修复

### ✉️ 核心 Prompt (User Request)
> “检查我的网页代码，彻底检查IID 经典标准误、HC1 稳健标准误、省份级单向聚类、省份与时间双向聚类、Wild Cluster Bootstrap等几个统计量的算法是否准确，因为Wild Cluster Bootstrap在小样本下出现了极高的拒绝率。”

---

### 💭 AI 核心思考与设计决策 (AI Design Decisions)

#### 1. 小样本下 Wild Cluster Bootstrap 退化的数理根源剖析
* **现象描述**：当组数极少（如 $G=6$）且仅有少数政策试点组（如 Treated = 1 个省份）时，网页端原本的 Wild Cluster Bootstrap 假阳性拒绝率竟然飙升至接近 100%，不仅没有起到“纠偏小样本”的作用，反而彻底崩溃。
* **数理死结**：原网页端的 Wild Cluster Bootstrap 采用了 **Unrestricted Wild Bootstrap**（非限制性残差，即基于 $Y_{it} = \hat{\beta} D_{it} + e_{it}$ 的估计残差 $\hat{e}_{it}$ 进行扰动）。
  - 在小样本且仅有 1 个组被处理的极端情形下，处理变量 $D_{dm, it}$ 的变异几乎 100% 集中于这一个被处理组中。
  - 普通最小二乘法 (OLS) 估计 $\hat{\beta}$ 时，会用这唯一的自由度去完美拟合这一个被处理组的均值。
  - 导致该处理组的 OLS 残差 $\hat{e}_{it}$ 被强制拟合至极度接近于 0！
  - 随后的 Bootstrap 过程中，该处理组的残差无论怎么乘以 $\pm 1$，由于其基数极度接近 0，其扰动项依然接近 0。由此，在所有自助迭代中，被处理组对 $\beta_b^*$ 的贡献几乎被锁死在 0，从而导致自助估计量 $\beta_b^*$ 的方差急剧塌陷，自助 $t$-统计量 $t_b^*$ 被死死锁在 0 附近。
  - 此时，样本 $t$-统计量 $|t_{cluster}|$ 很容易因为噪音越过 0，导致 $|t_b^*| \ge |t_{cluster}|$ 的概率（即 Bootstrap $p$-value）几乎为 0，零假设被 100% 拒绝！
* **设计决策（Restricted Wild Cluster Bootstrap）**：将算法重构为学术界黄金标准的 **Restricted Wild Cluster Bootstrap**（限制性残差，即在零假设 $H_0: \beta=0$ 约束下求残差）。
  - 在 $H_0$ 下，我们无需估计 $\beta$，因变量双向去均值后的 $Y_{dm, it}$ 即为限制性残差 $\tilde{e}_{it}$。
  - 在被处理组中，限制性残差没有被 OLS 拟合至 0，而是保留了原有的样本变异。
  - 自助估计量 $\beta_b^*$ 和 $t_b^*$ 由此恢复了健康的、符合真实有限样本抽样分布的波动，假阳性控制率重新完美收敛至 **5%** 名义水平！

#### 2. 标准误的学术级精度对齐
* **设计决策**：原网页中的聚类标准误没有包含 Stata 默认的有限样本自由度调整因子 $\frac{M-1}{M-k}$（其中 $M$ 为总样本量，$k$ 为回归参数数量）。
* **策略**：
  - 单向聚类纠偏乘子更新为 $q = \frac{G}{G-1} \times \frac{M-1}{df}$。
  - 年份维度聚类纠偏乘子同步更新为 $q_t = \frac{T}{T-1} \times \frac{M-1}{df}$。
  - 通过此数学对齐，彻底实现了网页前端的毫秒级 JS 仿真引擎与 Python（statsmodels）及 Stata 的回归精度完美契合。

## 📅 [2026-05-22] 实验沙盒多实验室暗金门户导航开发与一键部署

### ✉️ 核心 Prompt (User Request)
> “我的github里有很多这样的html文件，但是如何整成打开可以直接渲染成网页的版本呢？确保已经提交了github的库”

---

### 💭 AI 核心思考与设计决策 (AI Design Decisions)

#### 1. 解决 GitHub HTML 多文件快捷分发与直连渲染的工程痛点
* **问题描述**：GitHub 默认只显示源码且 MIME 类型为 `text/plain`，无法直接被浏览器解析渲染为动态 HTML 交互页面，如果每次都使用 GitHack 转换单个链接，对多文件管理非常繁杂。
* **设计决策**：
  - 指导用户启用官方 **GitHub Pages** 服务，以将整个仓库自动升级为静态 Web 服务器。
  - 在沙盒项目根目录下直接新建一个统一主导航文件 `index.html`。
* **物理路径设计**：
  - 将 `index.html` 写入 `empirical-playground/index.html` 根目录。
  - 所有链接均采用完美的**相对路径形式**（如 `labs/clt-demo/clt_dashboard.html`），这在本地和 GitHub Pages 静态托管中均能保持 100% 精确的导航跳转，无路径硬编码风险。

#### 2. 设计暗金玻璃拟物化中央大厅（Glassmorphism Multi-Lab Portal）
* **设计策略**：
  - **美学继承**：采用与主力 BDM 仿真看板一脉相承的高阶“黑金磨砂玻璃”艺术美学。背景基于深邃的太空黑（`#0b0c10`），叠加微小的金蓝色双核渐变发光粒子背景（`radial-gradient`）。
  - **信息层级结构**：使用 Cinzel 高档学术英文衬线体与 Noto Sans 现代中文，每张卡片均包含项目元分类（如 `Panel Econometrics`）、粗体实验名、通俗无 AI 腔的实验内核描述以及动态缓动箭头。
  - **动效微交互**：卡片悬浮时提供向上升腾 `-5px` 的 3D 位移、暗金描边高亮（`rgba(197, 160, 89, 0.6)`）与斜向流光扫掠动效，赋予学术工具极佳的视觉愉悦感与交互灵性。
