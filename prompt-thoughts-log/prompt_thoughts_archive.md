### 💭 AI 核心思考与设计决策 (AI Design Decisions)

#### 1. 物理目录结构设计
* **决策**：在工作区根目录下独立划分 `empirical-playground` 作为计量实验沙盒。不干扰原官方课件的 `empirical-methods-course` 和交互大看板 `interactive-simulations`。
* **原因**：这保证了项目边界清晰，其他人克隆 `empirical-playground` 时能作为独立的 Python-Jupyter 工作区进行运行。

#### 2. Jupyter 交互技术选型
* **决策**：采用 `ipywidgets` (滑块与选择器) 结合 `matplotlib` 渲染动态图像，而不使用复杂的 Shiny 或 Dash。
* **原因**：`ipywidgets` 可以完全嵌入在单个 `.ipynb` 笔记本中，轻量级、无需搭建复杂的本地 Web 容器，用户在 Jupyter Lab / VS Code 中双击即可体验，完美契合“一键配置、直观展示”的诉求。

#### 3. 语言风格调校 (去 AI 腔)
* **决策**：在所有实验的解读文档中，剔除多余的煽情、赞美、引申词汇（如“不是……而是……”、“极其深刻”、“惊人地”）。
* **策略**：
  * **用数理和逻辑推断说话**：解释为什么自相关会导致标准误被低估（因为自相关下，误差项包含了重叠的系统性冲击，OLS 将其视为独立样本，因而高估了有效样本量 $N$，进而低估了标准误差）。
  * **文字结构**：公式推导 $\to$ 模拟图表验证 $\to$ 核心思想启发（核心思想为“永远在政策分配层级聚类”）。

#### 4. 可追溯与接手机制
* **决策**：
  * 创建 `prompt-thoughts-log/` 文件夹。
  * 将用户的 Prompts 和模型的底层思考逻辑记录于本文件中 (`prompt_thoughts_archive.md`)。
  * 设立类似 Git commit-log 风格的物理文本文件 `operation_history.log`，记录操作人员和具体变更。
  * 在 `ai-coding-skills.md` 中增加“新 AI 助手一键配置与文档同步更新规范”，确保未来接手的 AI 能够实现自动化日志和 README 的同步迭代。

### 💭 AI 核心思考与设计决策 (AI Design Decisions)

#### 1. 蒙特卡洛（Monte Carlo）的直觉化降维教学
* **设计决策**：摒弃死记硬背的渐近大样本代数推导，改用**“计算机平行宇宙体检”**这一极具物理直觉的类比。
* **教学直觉**：
  - 用“造物主”的上帝视角向用户讲明白：我们在计算机内定死了政策真实效应为0（零假设 $H_0$ 为真）。
  - 然后模拟 1000 个平行宇宙。如果一个标准误公式是“诚实、正确”的，那么误诊率（拒绝率）应当且仅应当在 5% 左右。
  - 如果经典标准误拒绝率飙升至 45%，意味着它在 45% 的平行宇宙中“撒了谎”。

#### 2. 黑金玻璃拟物化网页实验室（Aesthetic Glassmorphism Dashboard）
* **设计决策**：放弃简单的 HTML 文本，采用顶尖现代前端 UI 设计规范：
  - **配色体系**：暗色背景，搭配磨砂玻璃面板，使用高度对比的暖金、靛蓝、翡翠绿、深绯红来对应各标准误类型。
  - **性能黑科技**：在前端用纯 JavaScript 实现高性能的 Within-Panel Panel OLS 面板双向去均值回归与组聚类方差估计。使数十万数据点的核密度计算与蒙特卡洛仿真在 JavaScript 内以毫秒级完成，拖动滑块时无需刷新网页、无需服务器，**SVG 图表进行流畅贝塞尔曲线过渡**。

#### 3. 经典与前沿图表的多维呈现
* **设计决策**：设计三大独立图表以实现直观具象化：
  - **拒绝率条形图**：展示三种标准误与 5% 名义警告线的偏离程度。
  - **t统计量经验分布拟合曲线**：使用 Epanechnikov 核密度估计（KDE）在 SVG 画布上动态绘制 $t$-统计量的经验 PDF 密度曲线，用户拉动 $\rho$ 到 0.8 时，能亲眼看到 OLS 的 t 分布曲线向两边“塌陷并长出巨大的胖尾”，越过 $\pm 1.96$ 的红色临界线。这以绝对完美的几何形式回答了“过度拒绝在数理上意味着什么”。
  - **置信区间覆盖体检图**：抽取 20 个平行宇宙的置信区间，若区间未包裹 0，则线段爆红（判定假阳性），直观展示区间覆盖率（Coverage Rate）如何随着自相关下降。
