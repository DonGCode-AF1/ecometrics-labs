## 📅 [2026-05-21] 项目初始化与 L1 BDM 仿真实验室设计

### ✉️ 核心 Prompt (User Request)
> “帮我生成一个新的项目文件夹并设计项目结构。这个文件夹的目的是这样：对于课件中涉及到的知识内容，提供可以用计算机进行实验、复现结果、或者提供图表等视图的选项，辅助我理解。设计实验代码的建议采用jupyter，可以在一个网页内快速调整代码并直观展示实验的结果，同时便于快速调整关键参数。对于实验的结果或者图表的结果，需要结合知识点给出清晰而明确的解读。（语言精炼，精准，提供核心的思想启发，以及实验的设计思路不要使用AI味道很重的表达，比如”不是而是“）。流程注意：1.对于过程中的prompt和思考类的回答，要放在一个专门的文件里留档，对于过程的代码，要有git版本管理、和操作日志管理，可追溯可协作；整体要便于上传到github，方便他人一键克隆。同时，要方便其他AI接手时可以通过框架、日志一键配置（确保skill有涉及文档更新的核心内容）。”

---

### 💭 AI 核心思考与设计决策 (AI Design Decisions)

#### 1. 物理目录结构设计
* **决策**：在工作区根目录下独立划分 `empirical-playground` 作为计量实验沙盒。不干扰原官方课件的 `empirical-methods-course` 和交互大看板 `interactive-simulations`。
* **原因**：这保证了项目边界清晰，其他人克隆 `empirical-playground` 时能作为独立的 Python-Jupyter 工作区进行运行。

#### 2. Jupyter 交互技术选型
* **决策**：采用 `ipywidgets` (滑块与选择器) 结合 `matplotlib` 渲染动态图像，而不使用复杂的 Shiny 或 Dash。
* **原因**：`ipywidgets` 可以完全嵌入在单个 `.ipynb` 笔记本中，轻量级、无需搭建复杂的本地 Web 容器，用户在 Jupyter Lab / VS Code 中双击即可体验，完美契合“一键配置、直观展示”的诉求。

#### 3. 语言风格调校 (去 AI 腔)
* **决策**：在所有实验的解读文档中，剔除多余的煽情、赞美、引申词汇（如“不是……而是……”、“极其深刻”、“惊人地”）。
* **策略**：
  * **用数理和逻辑推断说话**：解释为什么自相关会导致标准误被低估（因为自相关下，误差项包含了重叠的系统性冲击，OLS 将其视为独立样本，因而高估了有效样本量 $N$，进而低估了标准误差）。
  * **文字结构**：公式推导 $\to$ 模拟图表验证 $\to$ 核心思想启发（核心思想为“永远在政策分配层级聚类”）。

#### 4. 可追溯与接手机制
* **决策**：
  * 创建 `prompt-thoughts-log/` 文件夹。
  * 将用户的 Prompts 和模型的底层思考逻辑记录于本文件中 (`prompt_thoughts_archive.md`)。
  * 设立类似 Git commit-log 风格的物理文本文件 `operation_history.log`，记录操作人员和具体变更。
  * 在 `ai-coding-skills.md` 中增加“新 AI 助手一键配置与文档同步更新规范”，确保未来接手的 AI 能够实现自动化日志和 README 的同步迭代。
