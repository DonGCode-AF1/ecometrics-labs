一张图看四种准则

基于课程页第三部分“估计准则”：NLS、MLE、GMM、SMM。原课件中 NLS 使用了引力模型作为结构例子；其他方法这里补充经典直观案例。

NLS 让模型预测值尽量贴近观测值。适合模型直接给出条件均值的场景。
MLE 让实际观察到的数据，在模型设定下出现的概率最大。适合完整写出概率分布的场景。
GMM 不要求完整分布，只要求模型推出若干矩条件。适合有工具变量、正交条件或均衡条件的场景。
SMM 当理论矩无法解析计算时，先模拟模型，再让模拟数据的矩接近真实数据的矩。

共同逻辑：估计其实是在设计“距离”

四个方法的差别，不在于都用了优化器，而在于它们把“模型错了多少”定义成了不同的距离。

经济模型偏好、成本、约束、冲击
给定参数尝试一组 theta
模型预测均值、概率、矩或模拟数据
数据比较误差、似然、矩差
优化更新找到距离最小的 theta

NLS非线性最小二乘：让预测值贴近数据

一句话直觉：如果模型能告诉我们“给定参数时 y 应该是多少”，NLS 就寻找让预测误差平方和最小的参数。

theta_hat = argmin_theta sum_i [y_i - h(x_i; theta)]^2
怎么看：红点是数据，蓝线是模型预测。NLS 惩罚的是每个点到预测曲线的垂直距离。若误差方差大致稳定、目标是拟合条件均值，它非常自然。

案例：简化引力模型估计贸易成本弹性

课件中提到 Anderson-van Wincoop 引力模型。完整模型有多边阻力固定点；这里先用一个可直观运行的简化版本：贸易流随经济规模上升，随距离下降。

import numpy as np
from scipy.optimize import minimize

rng = np.random.default_rng(42)
n = 80
gdp_i = rng.lognormal(10.0, 0.7, n)
gdp_j = rng.lognormal(9.8, 0.8, n)
dist = rng.lognormal(6.0, 0.5, n)

# 真值：贸易流 = A * GDP_i^b1 * GDP_j^b2 * distance^(-tau) * noise
true = np.array([1.2, 0.85, 0.90, 1.15])  # logA, b1, b2, tau
log_trade = (
    true[0]
    + true[1] * np.log(gdp_i)
    + true[2] * np.log(gdp_j)
    - true[3] * np.log(dist)
    + rng.normal(0, 0.25, n)
)

def predict(theta):
    logA, b1, b2, tau = theta
    return logA + b1*np.log(gdp_i) + b2*np.log(gdp_j) - tau*np.log(dist)

def nls_objective(theta):
    resid = log_trade - predict(theta)
    return np.sum(resid**2)

res = minimize(nls_objective, x0=np.array([0.0, 1.0, 1.0, 1.0]))
print(res.x)
# 解释：tau 越大，距离对贸易流的抑制越强。

代码输出怎么读：预测贸易流贴近真实贸易流

理想形态：红点越贴近蓝色 45 度线，说明模型预测越接近观测贸易流。

经济解读：如果估计出的 tau 很大，图中远距离贸易对的预测值会被明显压低，表示距离或贸易成本对贸易流抑制更强。

结构估计提醒：完整引力模型中，贸易流不仅受双边距离影响，还受多边阻力项影响；若这些价格指数由固定点决定，NLS 外层每试一个参数，内层可能都要解一次固定点。

MLE最大似然：让观测数据最可能出现

一句话直觉：如果模型能写出每条数据出现的概率，MLE 就选择让整份样本“最像是从这个模型里生成出来”的参数。

theta_hat = argmax_theta sum_i log f(x_i | theta)
怎么看：MLE 不是只看均值误差，而是看数据落在整个概率分布中的“合理程度”。分布假设错了，似然也会把你带偏。

案例：二元选择 Logit 估计价格敏感度

消费者是否购买某产品。效用差为 beta0 + beta1 * price，购买概率是 sigmoid。MLE 选择最能解释 0/1 购买数据的参数。

import numpy as np
from scipy.optimize import minimize

rng = np.random.default_rng(7)
n = 500
price = rng.uniform(1.0, 5.0, n)
beta_true = np.array([2.0, -0.8])

def sigmoid(z):
    return 1 / (1 + np.exp(-z))

p_buy = sigmoid(beta_true[0] + beta_true[1] * price)
y = rng.binomial(1, p_buy)

def neg_log_likelihood(beta):
    p = sigmoid(beta[0] + beta[1] * price)
    p = np.clip(p, 1e-9, 1 - 1e-9)
    return -np.sum(y*np.log(p) + (1-y)*np.log(1-p))

res = minimize(neg_log_likelihood, x0=np.array([0.0, 0.0]))
print(res.x)
# beta1 通常为负：价格越高，购买概率越低。

代码输出怎么读：价格升高，购买概率下降

理想形态：绿色曲线是模型估计的购买概率，红点可以理解为不同价格区间的实际购买率。

MLE 的“拟合”不是画线：它是在最大化每个 0/1 选择被模型解释出来的概率。曲线位置和斜率都由似然函数决定。

适用场景：离散选择、拍卖出价分布、持续时间模型、需求模型中的选择概率。关键是你愿意相信概率分布或误差结构。

GMM广义矩估计：让模型矩接近数据矩

一句话直觉：如果模型不方便写完整概率，但能推出“某些平均关系应该等于 0”，GMM 就让这些平均偏差尽量小。

theta_hat = argmin_theta g_bar(theta)' W g_bar(theta), where g_bar(theta)=N^-1 sum_i g(Z_i, theta)
怎么看：GMM 比较的是样本中的平均正交关系。比如工具变量 z 应该和结构误差 xi 无关，所以 E[z * xi(theta)] = 0。

案例：用工具变量矩条件估计价格系数

需求里价格可能和未观测质量相关，OLS 会偏。若成本冲击 z 影响价格但不直接影响需求误差，可以用矩条件 E[z * residual] = 0。

import numpy as np
from scipy.optimize import minimize

rng = np.random.default_rng(11)
n = 800
x = rng.normal(size=n)
z = rng.normal(size=n)       # 工具变量：成本或距离冲击
xi = rng.normal(size=n)      # 需求冲击

# 价格受工具变量 z 和需求冲击 xi 共同影响，因此 price 内生
price = 1.0 + 0.7*z + 0.6*xi + rng.normal(scale=0.3, size=n)
y = 1.5 + 0.9*x - 1.2*price + xi

Z = np.column_stack([np.ones(n), x, z])

def moments(theta):
    b0, bx, alpha = theta
    resid = y - (b0 + bx*x - alpha*price)
    return (Z * resid[:, None]).mean(axis=0)

W = np.eye(3)

def gmm_objective(theta):
    g = moments(theta)
    return g @ W @ g

res = minimize(gmm_objective, x0=np.array([0.0, 0.0, 0.5]))
print(res.x)
# alpha 是价格敏感度；矩条件要求工具变量与结构残差正交。

代码输出怎么读：矩条件被压到接近 0

理想形态：估计后，每个柱子都应靠近蓝色 0 线。柱子越高，说明对应矩条件越没被满足。

经济解读：E[z * residual] = 0 是核心。它表示工具变量 z 中剩下的变化，不能系统性预测结构误差。

实践重点：GMM 的识别来自矩条件，不来自优化器。工具变量弱、矩条件错、权重矩阵乱用，都会让漂亮的数值结果失去经济意义。

SMM模拟矩估计：当矩只能模拟出来

一句话直觉：有些模型太复杂，理论矩没有封闭形式。SMM 就给定参数模拟一批“模型世界”的数据，再比较模拟世界和真实世界的统计特征。

theta_hat = argmin_theta [m_sim(theta)-m_data]' W [m_sim(theta)-m_data]
怎么看：左边是真实数据的矩，右边是模拟数据的矩。SMM 调参数，直到模拟出来的均值、方差、进入率、选择份额等尽量像真实世界。

案例：简化进入模型估计进入成本

一个市场是否有企业进入，取决于利润冲击是否超过进入成本。真实数据里只看到进入率和进入市场的平均利润，用 SMM 匹配这两个矩。

import numpy as np
from scipy.optimize import minimize

rng = np.random.default_rng(21)
n_market = 1000
true_cost = 1.1
profit_shock = rng.normal(loc=0.8, scale=1.0, size=n_market)
entered = profit_shock > true_cost
observed_profit = profit_shock[entered] - true_cost

m_data = np.array([
    entered.mean(),
    observed_profit.mean()
])

# 固定随机数，让目标函数更平滑
fixed_draws = rng.normal(loc=0.8, scale=1.0, size=5000)

def simulate_moments(cost):
    sim_entered = fixed_draws > cost
    sim_profit = fixed_draws[sim_entered] - cost
    return np.array([
        sim_entered.mean(),
        sim_profit.mean()
    ])

def smm_objective(theta):
    cost = theta[0]
    diff = simulate_moments(cost) - m_data
    return diff @ diff

res = minimize(smm_objective, x0=np.array([0.5]), bounds=[(0.0, 3.0)])
print(res.x)
# cost 越高，模拟进入率越低；SMM 用进入率和利润矩反推进入成本。

代码输出怎么读：真实矩和模拟矩逐步对齐

理想形态：同一组矩下，红柱和蓝柱越接近，说明当前参数模拟出的市场越像真实市场。

为什么有用：进入博弈、动态选择、随机系数需求这类模型常常难以写出解析矩，但可以模拟很多次。

课件提醒的延伸：如果每次目标函数评估都重新抽样，目标函数会抖动。实践中常固定随机抽样，使优化器看到的目标函数更平滑。

什么时候用哪个？

方法 你需要模型给出什么 典型场景 主要风险
NLS 条件均值或预测方程 非线性需求、引力模型、增长曲线 内生性、异方差、局部最优
MLE 完整概率密度或选择概率 Logit/Probit、拍卖、持续时间模型 分布假设错误、数值下溢、极端概率
GMM 矩条件或正交条件 工具变量、Euler 方程、资产定价矩 弱识别、错误矩条件、权重矩阵选择
SMM 可模拟的数据生成过程 进入博弈、动态选择、随机系数需求 模拟误差、计算成本、矩选择不充分

共同骨架

这四个方法都在做同一件事：定义一个“模型错了多少”的距离，然后找让距离最小的参数。真正困难的地方往往不是写优化器，而是选择能识别参数、能承受反事实解释的距离。

方法选择决策卡

能写出预测均值？ 优先考虑 NLS。先看残差图，再担心内生性和异方差。
能写出完整概率？ 优先考虑 MLE。检查分布假设、极端概率和数值稳定性。
只有正交条件？ 优先考虑 GMM。重点审查矩条件是否真的识别参数。
只能模拟模型？ 优先考虑 SMM。固定随机数，选择能识别参数的矩。
